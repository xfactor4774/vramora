How to use VRAMora

The Chart

üìç
Position: X-axis = system cost. Y-axis = max model you can run at Q4 (switchable via the Y-Axis dropdown above).
‚≠ï
Bubble size: Bigger = faster token generation at 7B Q4. A huge bubble means snappy responses.
üü¢
Bubble color: Green = power-efficient. Yellow = moderate. Red = power-hungry. Based on full-system TDP under LLM load.
üñ±Ô∏è
Hover a bubble to see full specs: cost, memory, speed, power draw, max model size, and notes.

Filters & Controls

‚òëÔ∏è
Sidebar filters let you narrow by hardware type (NVIDIA / Apple / System / Cluster) and memory tier.
üí∏
GPU-only cost: Check this if you already have a PC and just want to compare the GPU swap cost. Shifts the X-axis and recalculates value scores to reflect GPU cost only instead of full system cost.
üìä
Y-Axis dropdown: Switch between Max Model Size, VRAM, Token Speed, or Efficiency (t/s per Watt).
‚ú®
Highlight dropdown: Visually singles out the best value, fastest, most efficient, or most memory option from the current filtered set.

What Can I Run?

üîç
Pick a model from the dropdown and the chart dims any hardware that doesn't have enough memory to run it. Green-bright = you're good. Faded = OOM.

Table View

üìã
Switch to Table using the toggle top-right. All columns are sortable ‚Äî click a header to sort ascending, click again for descending.

Sharing

üîó
Share view encodes your current filters, Y-axis, and selected model into the URL. Copy and send it ‚Äî recipients land on exactly your view.

Data & Accuracy

üìù
All benchmarks are approximate ‚Äî sourced from hardware-corner.net, llama.cpp discussions, and community testing. Real-world numbers vary by model, quantization, and system config. Token speeds are for llama.cpp / MLX at 7B Q4_K_M unless noted.
üí∞
System costs include GPU + platform (CPU, mobo, RAM, PSU, case). Older cards (3060, 3080 Ti, 3090, A6000) are priced at current used market rates. Newer cards (4090, 5090) reflect street/MSRP. Treat all figures as ballpark estimates.
